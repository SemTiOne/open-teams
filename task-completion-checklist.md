# 任务完成检查清单模板

## 1. 适用范围

适用于当前工作空间内所有真实任务收尾场景，包括但不限于：

- `bugfix`
- `new-feature-dev`
- `security-fix`
- `performance-tuning`
- `api-refactor`
- `feature-refactor`

目标是把每次任务都收敛为统一闭环：

1. AI 自判是否完成
2. 用户确认是否完成
3. 输出复盘
4. 用户确认复盘
5. 同步更新相关资料

## 2. AI 完成判定检查清单

- [ ] 当前需求目标是否已经落地实现或修复
- [ ] 相关代码是否已经实际修改完成
- [ ] 改动范围是否与任务目标一致，没有无关漂移
- [ ] 关键验证是否已经执行
- [ ] 验证结果是否足以支撑“可交付完成”判断
- [ ] 是否仍存在阻塞项、未完成项、待联调项或显著风险
- [ ] 是否已经明确区分用户原有改动与本次任务改动

## 3. 面向用户的完成确认模板

- [ ] 明确说明 AI 当前判断是“已完成 / 基本完成但仍有遗留 / 未完成”
- [ ] 说明本次已交付的核心改动
- [ ] 说明涉及的关键代码路径
- [ ] 说明已执行的验证动作
- [ ] 说明验证通过项
- [ ] 说明未验证项
- [ ] 说明仍存在的风险
- [ ] 发起明确的用户确认

## 4. 复盘输出检查清单

- [ ] 问题或需求背景
- [ ] 根因或触发原因
- [ ] 最终解决方案
- [ ] 为什么选择该方案而不是其他方案
- [ ] 关键代码路径、接口、配置或运行链
- [ ] 本次验证方式与验证结果
- [ ] 本次踩坑点、误区或边界条件
- [ ] 后续如何避免同类问题再次出现

## 5. 资料同步检查清单

### 5.1 `docs/`

- [ ] 架构关系发生变化
- [ ] 模块职责认知发生变化
- [ ] 源码说明存在事实偏差
- [ ] 开发规范需要补充新的约束或避坑规则
- [ ] 接口说明、联调说明需要同步

### 5.2 `skills/`

- [ ] 某类场景已经形成稳定处理流程
- [ ] 某类问题出现了新的执行步骤或验证清单
- [ ] 某类常见坑需要沉淀到 `references/`
- [ ] 现有 `SKILL.md` 的约束或阅读顺序需要修正

### 5.3 `references/`

- [ ] 新补充了外围事实资料
- [ ] 数据结构、报文、字典类资料需要纠偏
- [ ] 需要补充现有文档无法承载的外围知识

### 5.4 索引清单

- [ ] 新增了可复用资产
- [ ] 脚本、文档、skill 入口发生变化
- [ ] 原有入口或优先级说明已失效

## 6. 最终闭环判定

- [ ] AI 已完成任务自判
- [ ] 用户已明确确认任务完成
- [ ] AI 已输出复盘
- [ ] 用户已明确确认复盘
- [ ] 相关资料已同步更新，或已向用户明确说明暂不更新的原因
