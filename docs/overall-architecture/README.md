# Overall Architecture

Place Architecture Decision Records (ADRs) and system design documents here.

## What Goes Here
- ADRs (Architecture Decision Records)
- System architecture diagrams and descriptions
- Technology stack decisions and rationale
- Cross-cutting design patterns

## Naming Convention
`YYYY-MM-DD-title.md` — e.g., `2026-01-15-use-postgres-for-primary-db.md`

## ADR Template
Copy from `TEMPLATE.md` in this directory. Each ADR should include:
- Context (why this decision is needed)
- Decision (what we chose)
- Alternatives considered
- Consequences (positive and negative)

## When to Write an ADR
- Choosing a new technology, framework, or service
- Significant changes to system architecture
- Decisions affecting multiple teams or projects
- Deprecating or replacing existing systems
