# Product Knowledge

Document product requirements, user stories, and domain knowledge here.

## What Goes Here
- Product requirement documents (PRDs)
- User personas and journey maps
- Feature specifications
- Competitive analysis
- Domain and industry knowledge

## Naming Convention
`feature-name.md` or `YYYY-MM-DD-topic.md`

## Template
Start each document with:
- **Purpose**: What problem does this solve?
- **Users**: Who benefits?
- **Scope**: What's in and what's out?
- **Success Metrics**: How do we measure success?

## Notes
- Keep documents focused — one topic per file
- Update when requirements change
- Link to related ADRs in `../overall-architecture/`
