# `docs/` 入口模板

本目录用于沉淀可被 AI 与协作者复用的文档资产。

首次采用模板时，不需要一次性补齐所有文档。推荐先按 [快速上手](../QUICKSTART.md) 跑通最小可用工作空间，再为当前最重要的项目补 `projects/` 入口。

建议至少包含以下子层：

- `overall-architecture/`：整体架构说明
- `projects/`：项目级源码说明与开发规范
- `development-specs/`：工作空间级治理规范
- `product-knowledge/`：业务理解与概念说明

使用要求：

1. 文档必须以真实源码和真实协作事实为依据
2. 先从目录入口进入，再按需读取具体文档
3. 文档不是源码替代物，事实冲突时以源码为准
4. 新工作空间优先补 `docs/projects/<project>/`，再按需要补整体架构、开发规范或业务知识
