# ☀️ 日报系统 — 每天 9:00 UTC+8

当心跳轮询触发时，检查当前时间。如果是 9:00-9:30 UTC+8（即 1:00-1:30 UTC），执行日报流程。

## 🔁 日报流程

### Step 1: 各角色 Agent 巡查

为以下角色分别生成快速巡查（各 spawn 一个子 Agent，mode=run，无上下文）：

- **仓库运营** — 检查 GitHub Stars/Issues/PRs/Discussions 有无新动态
- **内容进展** — 检查是否有新文章发布、社媒互动
- **社区动态** — Discord/微信群有无新成员、活跃情况
- **开发进展** — 仓库有无新 commit/合并/Release

### Step 2: 汇总报告

汇总后发送以下格式的日报：

```
📊 open-teams 日报 — 2026-MM-DD（周X）

**昨日进展：**
- [各模块具体进展]

**今日计划：**
- [按优先级排列的待办事项]

**关键数据：**
- ⭐ Stars: X (+Y)
- 👥 Contributors: X
- 💬 新 Issues/Discussions: X
- 📝 文章数据: 阅读 X，互动 X

**需要你决策：**
- [阻塞项或选择点]
```

### Step 3: 更新记忆

日报生成后写入 `memory/YYYY-MM-DD.md`。

---

## 📋 触发条件（心跳时逐项判断）

| 条件 | 行动 |
|------|------|
| 9:00-9:30 UTC+8 | 执行完整日报流程 |
| 其他时间 | 仅检查是否有紧急事项（新 Issue 含 bug 标签、安全报告等），有则通知否则 NO_REPLY |
| 23:00-08:00 UTC+8 | 除非紧急（安全漏洞），否则 NO_REPLY 不打扰 |
