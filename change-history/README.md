# change-history 目录说明

本目录用于记录已完成并经用户确认的任务级变更历史。

## 命名规范

- `change-history/<project>/YYYY-MM-DD-<task-type>-<slug>.md`

## 任务类型

| 类型 | 说明 |
|------|------|
| feature | 新功能 |
| bugfix | Bug 修复 |
| refactor | 重构 |
| docs | 文档变更 |

## 使用场景

当 AI 完成一个经你确认的任务后，可以在本目录记录：
1. 任务背景与目标
2. 实施过程关键决策
3. 最终结果与验收要点

这些记录帮助未来的 AI 会话理解项目演进历史。
