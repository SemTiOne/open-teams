# 场景 Skill 模板

## 适用范围

说明该 skill 面向的真实项目、模块、任务类型与不适用范围。

## 触发条件

- 列出应调用本 skill 的用户诉求、问题现象或开发阶段。
- 若当前任务还需要工作空间流程 skill，说明需要组合使用的流程阶段。

## 前置输入

- 必须先确认的需求范围、源码入口、配置事实与用户约束。
- 需要按需读取的项目文档、接口资料或运行环境信息。

## 首读资料

1. 对应项目源码说明文档
2. 对应项目开发规范文档
3. 与当前问题直接相关的本目录 `references/`
4. 仅在需要示例时读取本目录 `examples.md`

## 产物与退出条件

- 明确本 skill 需要形成的方案、实现、验证结果或资料同步建议。
- 明确何时应等待用户确认、移交下一阶段或判定该场景处理结束。

## 工作流程

1. 确认当前任务范围、所处阶段与需要组合的工作空间流程约束。
2. 按需读取项目文档和必要源码，形成以真实事实为依据的判断。
3. 输出方案并等待用户明确确认；未确认前不进行实现性修改。
4. 用户确认后，依据已落地的实施计划处理当前节点。
5. 执行与当前改动范围匹配的验证，记录结果与遗留风险。
6. 按节点或任务状态汇报结果，并按工作空间约束等待验收及判断资料同步。

## 验证清单

- 需求或问题范围是否得到证据支持。
- 实际变更是否只落在确认范围内。
- 关键命令、测试或人工验证是否已经执行并记录结果。
- 是否说明了未覆盖项、风险与后续资料同步建议。

## 禁止事项

1. 不基于猜测补全事实
2. 不绕过方案确认直接实现
3. 不忽略用户已有改动
4. 不在缺少验证证据时宣称节点或任务完成

## 按需资料

- `references/`：记录当前项目和场景直接相关的约束、接口边界与常见问题。
- `examples.md`：记录可复用的输入表达、执行产物或验收示例。
